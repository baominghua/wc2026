# 赛后复盘与公开数据源设计

日期：2026-06-18

## 目标

在每场比赛结束后，把赛前预测结果与真实赛果进行对照，生成单场复盘报告和整体准确率统计。复盘结论会以很小的权重回写到下一轮预测，用来体现球队即时状态、小组出线形势、平局动机和预测偏差修正。

## 数据原则

项目当前不使用商用接口。所有客观事实数据必须来自公开、免费或用户显式配置的合规来源，例如 ESPN 公共比分端点、FIFA 官方排名页、公开赛程页或用户自有的免费额度接口。拿不到的字段不再用样例、手工 JSON 或预测值补假数据，而是明确标记为未取得公开数据。

Opta、WhoScored、Stats Perform 等数据只作为未来授权接入方向，不内置未授权抓取。

## 功能范围

- 单场复盘：展示赛前主预测、另外两个高概率比分、实际比分、胜平负命中、比分命中、冷门命中、偏差原因、下一次预测注意事项。
- 整体统计：统计 1 选准确率、2 选准确率、3 选准确率、比分 1 选准确率、比分 3 选准确率、冷门准确率。
- 下载能力：整体统计支持 CSV 下载，单场复盘支持 Markdown 报告下载。
- 后续权重：根据已完赛事实生成 review adjustment，影响下一轮预测中的进攻状态和小组形势平局倾向。
- 用户入口：新增“复盘”页面，桌面端和移动端均可从主导航进入。

## 数据流

1. 后端合并公开实时赛程与本地基础赛程。
2. 对已完赛比赛调用预测模型生成无复盘权重的基准预测。
3. 复盘引擎比较预测和真实比分，生成准确率与偏差说明。
4. 当用户预测后续比赛时，预测 API 根据已完赛同组比赛计算轻量复盘权重。
5. 前端复盘页读取 `/api/v1/reviews/`，下载统计时读取 `/api/v1/reviews/export.csv`。

## 边界

- 复盘权重只做小幅校准，不直接覆盖基础模型。
- 若没有技术统计，偏差原因会降级为比分、胜平负、进球效率和小组形势分析。
- 若公开源没有助攻、角球、牌面等字段，页面必须显示未取得公开数据，不得生成看似真实的数据。

## 验收

- 后端复盘单元测试通过。
- 预测 API 能在第二轮比赛中带上复盘权重说明。
- 前端 production build 通过。
- 复盘页能加载统计、切换单场报告、下载 CSV 和 Markdown。
- 文档说明公开免费数据源约束、复盘权重和下载内容。
