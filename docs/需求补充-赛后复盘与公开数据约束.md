# 需求补充：赛后复盘与公开数据约束

## 背景

用户要求项目中的客观事实数据必须实时、自动、公开可追溯，不再出现默认假数据、刷新后重复计算、手工维护榜单等问题。用户当前没有商用接口，因此数据方案必须基于公开免费来源或用户自行配置的免费额度接口。

## 新增需求

1. 增加“复盘”模块。
2. 每场比赛结束后，对照预测结果和真实结果生成复盘报告。
3. 复盘报告解释预测偏差，并给出下一次预测要注意的因素。
4. 复盘结论以小幅权重影响后续预测，重点考虑第一轮后的状态、小组出线形势和平局动机。
5. 增加整体预测准确率表，统计 1 选、2 选、3 选、比分和冷门命中率。
6. 单场复盘报告和整体统计表均支持下载。
7. 所有客观数据均来自公开免费数据源；拿不到的数据不得伪造。

## 非目标

- 不内置未授权的商用数据抓取。
- 不把预测值写入赛果、积分、射手榜、助攻榜、牌面或阵容。
- 不把手工 JSON 作为线上默认事实源。

## 验收标准

- 已完赛比赛能在复盘页出现预测与真实结果对照。
- 预测接口能返回复盘校准因素。
- 下载文件包含可读的复盘与统计内容。
- 后端测试和前端构建通过。
- 文档说明数据来源边界与降级策略。
